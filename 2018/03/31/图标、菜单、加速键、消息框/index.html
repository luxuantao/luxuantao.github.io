分别使用windows SDK和wxpython制作一个简易界面，显示图标、菜单、加速键、消息框。

windows SDK版本

wxpython版本

相比两者，显然wxpython比windows编程方便，而且windows编程需要你先用微软的软件（如visual studio）生成.rc资源文件和resource.h文件，然后在cpp文件中进行调用，但wxpython完全是用代码动态添加菜单，加速键之类的东西，省了很多事。

PS:

visual studio就装了我好久，这软件太霸道了，必须装C盘，结果装不下。。。于是我开始自己写.rc文件和resource.h文件，写了一天，结果还是放弃，因为像加速键之类的东西我压根就不知道在.rc文件里到底长什么样。多亏找到一个软件能重新分区，给C盘腾了地方，装好vs后，半小时解决战斗。

按理说，wxpython在windows和linux都应该能运行（事实也确实如此），但我把windows上成功运行的wxpython代码放在ubuntu上跑的时候，菜单栏不见了！我以为是代码的问题，找了半天没发现，其实压根不是代码的问题，ubuntu下的程序菜单是在桌面顶部的桌面菜单上，只不过你把鼠标放上去它才会显示。。。
